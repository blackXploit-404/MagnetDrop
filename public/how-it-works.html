How It Works

Understand the process behind MagnetDrop.

  • Client connects via WebSocket – When a user initiates a connection, they can send a request to share a magnet link.
  • Magnet link validation – The server checks if the provided magnet link is in the correct format. If it's invalid, an error message is sent.
  • Short URL generation – If valid, the server generates a short ID and stores the link temporarily.
  • Expiration mechanism – The link remains active for 1 minute, after which it is automatically deleted.
  • Accessing the shared link – Anyone with the short URL can retrieve the magnet link within the expiration time.

Features

  • WebSocket-based communication for real-time sharing.
  • Rate limiting to prevent abuse (max 10 requests per minute).
  • Temporary storage mechanism to auto-delete expired links.
  • Simple redirect system for retrieving magnet links.
